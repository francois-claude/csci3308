Docker always runs as root, which means you will have to use sudo whenever running docker. Alternatively, you can create a user group to ditch the use of sudo. For now, just remember to prepend all of the docker commands such as docker run with sudo.

Part A: Section 2. Introduction to Docker

(15 Minutes)

1. Running the Hello World Container
Open the terminal and run the following command
docker run hello-world

Hello from Docker!
This message shows that your installation appears to be working correctly.

To generate this message, Docker took the following steps:
 1. The Docker client contacted the Docker daemon.
 2. The Docker daemon pulled the "hello-world" image from the Docker Hub.
    (amd64)
 3. The Docker daemon created a new container from that image which runs the
    executable that produces the output you are currently reading.
 4. The Docker daemon streamed that output to the Docker client, which sent it
    to your terminal.

To try something more ambitious, you can run an Ubuntu container with:
 $ docker run -it ubuntu bash

Share images, automate workflows, and more with a free Docker ID:
 https://hub.docker.com/

For more examples and ideas, visit:
 https://docs.docker.com/get-started/
Copy
You'll likely see information about the image being pulled if it's not cached locally. This is normal, and will happen every time you run a docker image for the first time. If all goes well the above should be visible at the end of the output.
2. Listing containers
Run the following command in terminal to see the containers on your system 
docker ps -a
CONTAINER ID   IMAGE         COMMAND    CREATED         STATUS                     PORTS     NAMES
7164fe3c9664   hello-world   "/hello"   3 minutes ago   Exited (0) 3 minutes ago             great_kare
Copy

Note that docker ps by default only lists the running containers (a container is an instance of a docker image). We used the flag -a to lists all containers.
3. Removing stopped containers
Removing old containers is a vital part of using docker. One way to this is by removing stopped containers such as the one reported above. 
docker rm <container_name> /* replace with your container name */
Copy
For example, here we're using the name "great_kare". Look at the "NAMES" column in the snippet in part 2 of this section. This name is automatically generated by docker, and yours will likely be different. 
docker rm great_kare
Copy
Checking our containers again, and we can see that it has indeed been removed. 
docker ps -a
CONTAINER ID   IMAGE         COMMAND    CREATED         STATUS                     PORTS     NAMES
Copy
4. Removing containers automatically on exit
We can also use the flag --rm to remove the container automatically once it finishes execution. 
docker run --rm hello-world
Copy
You should see the same output as before. However, if you check for containers using docker ps -a, you'll notice it has already been removed. Wonderful!
5. Running an interactive container
You might've noticed that the hello-world image mentioned an "ubuntu" image. What gives, that's an operating system? Let's give it a go. 
docker run -it ubuntu bash
Copy
There are some new parts to this one, so let's break it down. The flag -it tells docker you'd like to run the image interactively, and that you'd like a tty shell (it's two flags really, but you'll almost always use them together). The bash indicates that we'd like to use bash as our command.
Your command prompt should have changed a bit. Notice below that the user and host has changed.
Copy
We'll be using this difference in prompt to indicate when the command is being executed in docker. Note that the host (be190571a423) is not likely to match on your system. That is because the host here is the generated container ID.
You can exit the container as follows: 
exit
Copy
Don't forget to clean up the container you just created! Remember, you can avoid this by using the flag --rm.
5. Running an detached container
As well as being able to run containers interactively, we can run them in detached mode. 
docker run -dt ubuntu bash
ace17ba3269cab8cf250c57841ab216f14963715ff646607be90af17a41a6ef6
Copy
We are now detached from the container but it's still running. We can check using: 
docker ps
CONTAINER ID   IMAGE     COMMAND   CREATED         STATUS         PORTS     NAMES
ace17ba3269c   ubuntu    "bash"    2 minutes ago   Up 2 minutes             magical_thompson
Copy
We'll see why detached containers are useful soon, but for now we'd like to stop this one. Since it is running, our previous methods won't work. 
docker rm magical_thompson /* replace with your container name */
Error response from daemon: You cannot remove a running container ace17ba3269cab8cf250c57841ab216f14963715ff646607be90af17a41a6ef6. Stop the container before attempting removal or force remove
Copy
As docker informs us, we must first stop the container, or we can force remove. Let's force remove the container. 
docker rm -f magical_thompson /* replace with your container name */
Copy
6. Wrapping up
Did you leave any containers? Make sure to double check! You've got the basics of docker down, and feel free to come back to this section as you need. As a tip, if you forget a command or flags, docker's built in help is very useful.
For top level info: 
docker --help
Copy
Or for command specific information: 
docker COMMAND --help
Copy

Part A: Section 4. Pull & Test PostgreSQL

(15 Minutes)

1. Pull and run PostgreSQL
docker run -dt -v postgres-data:/var/lib/postgresql/data -e POSTGRES_PASSWORD=password postgres:latest
3c2ad758d53767ae728a1de868f784af97a8c693b145465c49c0f37271386490
Copy
New flags:
  • -v postgres-data:/var/lib/postgresql/data: bind a volume "postgres-data" to /var/lib/postgresql/data. This volume will now be preserved across subsequent runs of postges.
  • -e POSTGRES_PASSWORD=password: Set an environment variable "POSTGRES_PASSWORD" within the container.
  • postgres:latest: Different versions of containers can be specified by tags, this tells docker that we want the latest postgres image.
2. Check the image is running

Part A: Section 5. Run & Test Node.js

(10 Minutes)

1. Create practice Node.js website
 vim test_server.js
Copy
Copy into your file the following code. As a note, you can also use the built-in text editor instead of vim/nano to modify the text files. However, it is still strongly encouraged for you to become familiar with a terminal text editor. Not all systems have a desktop to work within! 
const http = require('http');

const hostname = '0.0.0.0';
const port = 3000;

const server = http.createServer((req, res) => {
  res.statusCode = 200;
  res.setHeader('Content-Type', 'text/plain');
  res.end('Hello World\n');
});

server.listen(port, hostname, () => {
  console.log(`Server running at http://${hostname}:${port}/`);
});
Copy
2. Start your Node.js website in a docker container
 docker run -dt -v $(pwd):/app -p 3000:3000 node:12-alpine node /app/test_server.js
Copy
If you are using a windows OS use the following command: 
 docker run -dt -v "%cd%":/app -p 3000:3000 node:12-alpine node /app/test_server.js
Copy
New flags:
  • -v $(pwd):/app (or) -v "%cd%":/app: Bind the current directory (on your local machine) to /app in the container. This is great for development, as everything you change in the working directory of your host machine will be reflected in the docker container
  • -p 3000:3000: Map port 3000 of the host to port 3000 in the container.
  • node /app/test_server.js Run our script in the container.
Now check that it works by browsing to http://127.0.0.1:3000 in Chrome.
