Part A: Section 3. Make an API Request

(20 Minutes)

In this section, we will demonstrate how your server makes a request to the NYTimes API

1. Make a NYTimes API Request with Axios

Axios is a lightweight HTTP client that provides a bunch of utility functions to make GET and POST requests to different web services. Axios can make a request to a given URL and will return a response object. By default, Axios' response object is a JSON object.


axios({
  url: `https://api.nytimes.com/svc/movies/v2/reviews/search.json?query=${title}&api-key=${api_key}`,
    method: 'GET',
    dataType:'json',
  })
  .then(items => {
    // Send some parameters
  })
  .catch(error => {
    // Handle errors
  })
});
Copy
url : ``
We use a pair of backticks `` to build the request url. This url comprises the resource URL for searching movie reviews, https://api.nytimes.com/svc/movies/v2/reviews/search.json, and a number of search parameters which we have appended to the url. Backticks are template literals or string literals that allow embedded expression (i.e. variables). You can read more about template literals here.
method: 'GET'
The NYTimes API specifies that this request must be a GET request. Remember, GET requests carry the request parameters in the url.
dataType: 'json'
We want the return object to be JSON because it is easy to parse.
.then(items => {
The function that executes if the request was successful and returns a response object.
.catch(error => {
The function that exectues if the request fails (for whatever reason) and returns and error.

Part B: Section 2. Reading the Flickr API docs

  • The documentation for the Flickr API can be found here.
  • You will be using the Flickr Photos search API to fetch images for your website. The Flickr search API return a list of photos matching some criteria. Its documentation can be found here.
  • The search criteria for the API can be specified using the API arguments. The decription and usage of the arguments can be found in the Flickr search API documentation page linked above.

    API Arguments to be used for this homework

    • api_key : This is the only mandatory argument. Please use the api_key obtained in Part 1.
    • tags : Use this to filter the photos based on tags.
    • extras : Use this to retrieve image_url of the photo. For example, if you need the API to also return a link to the image, set the extras argument to url_sq.
    • privacy_filter : Use value 1 to access only public photos.
    • safe_search : Use value 1 to access only safe photos.
    • page : The page to return.
  • In order to explore the API and play around with its arguments, you can use the search API explorer provided by Flickr. You can use this to build the required url for the API.
  • Building url using API explorer

    • Prepare your request in the explorer by providing the required arguments.
    • Make sure that you set the output type as json in the API explorer.
    • Select the the "Do Not Sign Call" checkbox and click on the "Call Method" button.
    • At the bottom of the page you should be able to find the completely formed url based on your arguments.
